Stopping the TOOLBOX

The Toolbox can be stopped stopping all the components.

Stopping Apache on a Windows machine

This step has to be performed only if the Apache Server has been used as a proxy server.

Apache is assumed installed as a service. After the installation Apache is running in the background and will start automatically when the machine boots. Apache stopping can be controlled in four ways:

  • by the Start menu, selecting "Start"-->"Programs"-->"Apache HTTP Server 2.0.43"--> "Control Apache Server" and finally "Stop".
  • by clicking on the "Apache Monitor" icon on the Windows Tool bar and selecting "Stop".
  • by opening the "Services" window (in the "Control Panel" on Windows NT, in "Control Panel" --> "Administration tools" on Windows 2000), selecting "Apache2", and then clicking on Stop.

Stopping Apache on a Linux machine

To stop and restart Apache by the apachectl script, type one of the following:

  • $ apachectl stop (STOP NOW)
  • $ apachectl graceful (GRACEFUL RESTART)
  • $ apachectl restart (RESTART NOW)

Stopping Tomcat on a Windows machine

To stop Tomcat, in a dos shell run:

  • $CATALINA_HOME\bin\shutdown.bat

where $CATALINA_HOME is the full path to the Tomcat installation directory

Starting from TOMCAT 5.0 the shutdown.bat scripts is no more provided. Thus the server can be stopped via the shortcut Monitor Tomcat in Console placed to Start Menu --> Programs --> Apache Tomcat 5.X --> Monitor Tomcat. This will add an icon in the toolbar. Double click on the new icon and stop the server by clicking on the Stop button.

Stopping Tomcat on Linux machine

To stop Tomcat, perform:

  • $ cd Root_Directory_Of_Tomcat /bin
  • $ shutdown.sh

where Root_Directory_Of_Tomcat is the Tomcat installation directory.
