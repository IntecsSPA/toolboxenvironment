Store processing

The store processing aims to provide the storing functionalities accessible through a WPS interface. This processing extract information from the input message, fuse it with the configuration values and then invoke the ARMS servlet. No special processing is performed by it. For a full description of how the store operation works in general, please refer to this section The store processing is accessible at the following URL: http://<TOMCAT_IP>:<TOMCAT_PORT>/TOOLBOX/services/<SERVICE_NAME>
The SOAP Action associated to this processing is ExecuteProcess_store

Input message

The input message is an XML document which follows the WPS interface rules. The fields are described in the table below:

Field

Type

Mandatory

Description
InputDatatType Enumeration Yes The data type of the input file. Valid values are:
  • eo
  • iso
Metadata URL No This field contains a link (URL) to metadata
DataDuration Integer No This field contains an amount of time, expressed in millisecond, after which data is removed from the ARMS
OutputDataType Array Yes This is a list of protocols under which data and metadata are published.

Output message

The Output message is an XML document which follows the WPS interface rules. The fields are described in the table below:

Field

Type

Mandatory

Description
Identifier Integer Yes This field constains the transaction identifier associated to the request.
