Setup a rolling archive

The ARMS has been developed with the rolling archive feature in mind. Through this, the ARMS is capable of automatically delete data and metadata after a configurable amount of time. Setting up a rolling archive is simple and requires only a proper configuration of the Archiving Servlet. To enable the rolling archive the administrator has to edit the configuration file (see here) and has to provide a value for the field delete.after. When this field is set, the ARMS acts as a rolling archive, deleting automatically all stored data after the specified amount of time. Data and metadata can still be deleted via the delete operation.
