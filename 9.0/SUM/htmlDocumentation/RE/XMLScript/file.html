File handling

Tag Name dumpFile
Attributes
Name Type Use Default Description
fileType     text Defines the type of the file to be stored: the possible values are text and binary.
Description This tag dumps, in a file whose name is given by the result of the first child element, the object given by the second child element. It returns null. The fileType attribute specify the type of the file to be stored. By default the file type is text (thus the object in the second child element is treated as a string).
Examples 
<dumpFile>
	<string>C:\MyData\data01.txt</string>
	<string>This is my file content.</string>
</dumpFile>
dumps the string "This is my file content." in the file C:\MyData\data01.txt file and returns null. 
<dumpFile fileType="binary">
	<string>C:\MyData\target.jpg</string>
	<loadFile fileType="binary">
		<string>C:\MyData\source.jpg</string>
	</loadFile>
</dumpFile>
Copies the binary file C:\MyData\source.jpg into the file C:\MyData\target.jpg and returns null. 
<dumpFile>
	<stringCat>
		<variable name="directoryPath"/>
		<string>file.txt</string>
	</stringCat>
	<variable name="fileContent"/>
</dumpFile>
Dumps in the file "file.txt" under the directory whose path is stored in the variable "directoryPath" the string contained in the variable "fileContent".

Tag Name fileDelete
Description This tag deletes a file whose name is given by the enclosed element. Returns true if and only if the deletion was possible (for example it returns false if the file didn't exist).
Examples 
<fileDelete>
	<string>C:\Dir\file.txt</string>
</fileDelete>
Deletes the file stored in the C:\Dir\file.txt 
<fileDelete>
	<stringCat>
		<string>C:\Dir\</string>
		<variable name="fileName"/>
	</stringCat>
</fileDelete>
Deletes the file whose name is stored in the variable "fileName" and is contained in the the directory C:\Dir.

Tag Name fileExists
Description Returns a boolean value corresponding to the existence of the file whose name is given by the enclosed element.
Examples 
<fileExists>
	<string>C:\Dir\file.txt</string>
</fileExists>
If the file C:\Dir\file.txt exists returns true, otherwise returns false. 
<fileExists>
	<stringCat>
		<string>C:\Dir\</string>
		<variable name="fileName"/>
	</stringCat>
</fileExists>
If the file contained in the directory "fileName" and whose name is stored in the variable "fileName" exists returns true, otherwise returns false.

Tag Name loadFile
Attributes
Name Type Use Default Description
fileType     text The addressed host
Description This tag loads a file whose name is given by the result of the enclosed tag. It returns the file content according to the fileType attribute value: a string if fileType=text or a binary stream if fileType=binary. By default it returns a string.
Examples 
<variable name="fileContent">
	<loadFile>
		<string>C.\Dir\File.txt</string>
	</loadFile>
</variable>
Loads the file C:\Dir\File.txt and stores its contents in the variable "fileContent".

Tag Name mkdir
Description This tag creates the directory whose name is given by the enclosed element. This works also if the parent directory doesn't exist: the whole path is created
Examples 
<mkdir>
	<string>C:\ParentDir\Dir</string>
</mkdir>
Creates the directory Dir (and possibly the parent directory ParentDir) and returns true if and only if the directory was created, along with all necessary parent directories; false otherwise .

Tag Name rmdir
Description This tags removes recursively a directory and all its contents. The name of the directory to be removed is given by the enclosed element.
Examples 
<rmdir>
	<string>C:\ParentDir\Dir</string>
</rmdir>
Removes the directory Dir (and recursively all its content) and returns null.

Tag Name path
Attributes
Name Type Use Default Description
startFrom xsd:string optional  

The directory to start from. This attribute may take the following values:

  • TMP (for system temporay dir)
  • HOME (for home directory)
  • SERVICE_RESOURCE_DIR (for the Resources dir inside the service directory
  • SERVICE_COMMON_SCRIPTS_DIR (for the Common Scripts dir inside the service directory
Description This tag returns a path string that is composed by the concatenation of the starting dir (provided optionally by the startFrom tag) and the computation of all the children tags. If not provided through that startFrom attribute, the start point can be set through the children attribute. The full path is returned using the system path separator (/ for Linux machines, \ for windows ones)
Examples 
<setVariable name="path">
	<path startFrom="HOME">
		<string>/my_dir/</string>
		<string>file.jar</string>
	</path>
</setVariable>

Returns the full path of the file my_dir/file.jar located under the user home diretory. 

<setVariable name="path">
    <path>
       <string>c:\temp\</string>
       <string>file.jar</string>
    </path>
</setVariable>

Returns the path c:\temp\file.jar. WARNING: If used under Linux this script snippet will return the following: c:/temp/file.jar.

Tag Name fileInfo
Attributes
Name Type Use Default Description
information xsd:string required   The information type to be returned. Valid values are: MODIFICATION and LENGTH
Description Returns information about the file described by the child tag.
Examples 
<fileInfo information="MODIFICATION">
		<string>C:\Dir\File.txt</string>
</fileInfo>

Returns the modification date associated to the file.

<fileInfo information="LENGTH">
		<string>C:\Dir\File.txt</string>
</fileInfo>
Returns the length of the file File.txt.
