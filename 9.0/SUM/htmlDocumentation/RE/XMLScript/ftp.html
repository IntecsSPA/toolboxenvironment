FTP

Tag Name ftpAccount
Attributes
Name Type Use Default Description
writePermission xsd:boolean   false The write permission for the user account.
duration com:ToolboxTime required The duration of the account, expressed by a non-negative number followed by a letter (s=second m=minute h=hour d=day w=week). It may be expressed using the $variable syntax
Description creates a new account on the Toolbox FTP server, with a mandatory duration. After the duration time expires, the Toolbox automatically remove the ftp account (but does not delete any file). It is possible to perform an action at user expiration filling the onExpire tag with a script snippet. The FTP user is given by the enclosed element of the "user" child element, the password is given by the enclosed element of the "password" child element, the root directory of the account is given by the enclosed element of the "rootDir" child element, the duration of the account is given by the value of the "duration" attribute. The duration is expressed by a non-negative number followed by a letter (s=second m=minute h=hour d=day w=week).
Examples 
<ftpAccount duration="5d">
   <user>
      <string>username</string>
   </user>
   <password>
      <string>xbshj</string>
   </password>
   <rootDir>
      <string>C:\Root\Dir</string>
   </rootDir>
   <onExpire> 
      <sequence> 
         <log info="user expired"/">
      </sequence>
   </onExpire> 
</ftpAccount>
Creates an FTP account on the TOOLBOX FTP server with the following parameters:
  • expiration time: 5 days
  • user name: username
  • password: xbshj
  • Root directory: C:\Root\Dir

    • WARN: the created accounts have solely read permission, NOT write permission.

    Tag Name ftpDelete
    Attributes
    Name Type Use Default Description
    pasv xsd:boolean required false It selects among the active (default) or the passive mode transfer.
    Description This tag performs an FTP delete operation. The "pasv" attribute selects among the active (default) or the passive mode transfer. The "host" child contains an element that gives the name of the host to be contacted as FTP server. The "port" child contains an element that must return an integer giving the FTP port. The user child contains an element that must return the user name. The password child contains an element that must return the password. The remotePath child contains an element that must return the remote path of the file to be deleted. This tag returns true if it was possible to delete the file.
    Examples 
    <ftpDelete>
    	<host>
    		<string>host.name</string>
    	</host>
    	<port>
    		<literal value="2121"/>
    	</port>
    	<user>
    		<string>username</string>
    	</user>
    	<password>
    		<string>hrjrj</string>
    	</password>
    	<remotePath>
    		<string>tmp/remote/path/file.txt</string>
    	</remotePath>
    </ftpDelete>
    Deletes via FTP the file "file.txt" under the remote directory "tmp/remote/path" of the host "host.name". Connection is established using the port "2121" and login parameters: "username", "hrjrj". If deletion succeeds true is returned.

    Tag Name ftpGet
    Attributes
    Name Type Use Default Description
    transfer     ascii The transfer mode. It takes values: ascii (default) and binary
    pasv xsd:boolean required false It selects among the active (default) or the passive mode transfer.
    Description This tag performs an FTP get operation. The "transfer" attribute (ascii/binary) selects the transfer mode. The "pasv" attribute selects among the active (default) or the passive mode transfer. The "host" child contains an element that gives the name of the host to be contacted as FTP server. The "port" child contains an element that must return an integer giving the FTP port. The "user" child contains an element that must return the user name. The "password" child contains an element that must return the password. The "remotePath" child contains an element that must return the remote path of the file or directory to be got. The "localPath" child contains an element that must return the local path where to store the downloaded data.
    Examples 
    <ftpGet transfer="binary">
    	<host>
    		<string>host.name</string>
    	</host>
    	<port>
    		<literal value="2121"/>
    	</port>
    	<user>
    		<string>username</string>
    	</user>
    	<password>
    		<string>hrjrj</string>
    	</password>
    	<remotePath>
    		<string>tmp/remote/path/file.tar</string>
    	</remotePath>
    	<localPath>
    		<string>/local/path/file.tar</string>
    	</localPath>
    </ftpGet>

    Downloads via FTP the file "file.tar" stored on the FTP server hosted on the machine "host.name" and listening on port 2121. The login parameters are (username, hrjrj). The file is stored on the local machine under the directory "/local/path".

    <ftpGet transfer="binary">
    	<host>
    		<string>host.name</string>
    	</host>
    	<port>
    		<literal value="2121"/>
    	</port>
    	<user>
    		<string>username</string>
    	</user>
    	<password>
    		<string>hrjrj</string>
    	</password>
    	<remotePath>
    		<string>tmp/remote/path/</string>
    	</remotePath>
    	<localPath>
    		<string>/local/path/</string>
    	</localPath>
    </ftpGet>
    Downloads via FTP the content of the directory path stored on the FTP server hosted on the machine "host.name" and listening on port 2121. The login parameters are (username, hrjrj). The data is stored on the local machine under the directory "/local/path".

    Tag Name ftpPut
    Attributes
    Name Type Use Default Description
    transfer     ascii The transfer mode. It takes values: ascii (default) and binary
    pasv xsd:boolean required false It selects among the active (default) or the passive mode transfer.
    Description This tag performs an FTP put operation. The transfer attribute (ascii/binary) selects the transfer mode. The "pasv" attribute selects among the active (default) or the passive mode transfer. The "host" child contains an element that gives the name of the host to be contacted as FTP server. The "port" child contains an element that must return an integer giving the FTP port. The user child contains an element that must return the user name. The password child contains an element that must return the password. The remotePath child contains an element that must return the remote path where to store the file. The localPath child contains an element that must return the local path of the file to be sent.
    Examples 
    <ftpPut transfer="binary">
    	<host>
    		<string>host.name</string>
    	</host>
    	<port>
    		<literal value="2121"/>
    	</port>
    	<user>
    		<string>username</string>
    	</user>
    	<password>
    		<string>hrjrj</string>
    	</password>
    	<remotePath>
    		<string>remote/path/file.tar</string>
    	</remotePath>
    	<localPath>
    		<string>/local/path/file.tar</string>
    	</localPath>
    </ftpPut>
    Transfers via FTP (binary mode) the file "file.tar" stored under the local directory "/local/path" on the FTP server hosted on the machine "host.name" and listening on port 2121. The login parameters are (username, hrjrj). The file is stored on the remote machine, in the file named file.tar in the directory "remote/path/" (NOTE:The file name on the remote machine must be specified. Further, the directory path on the remote machine where the file will be stored must be already existent at transfer time).

     

    Tag Name ftpRename
    Description

    This tag performs a rename operation over a file stored in an ftp. The "host" child contains an element that gives the name of the host to be contacted as FTP server. The "port" child contains an element that must return an integer giving the FTP port. The user child contains an element that must return the user name. The password child contains an element that must return the password. The remotePath child contains an element that must return the remote path of the file to rename. The newRemotePath child contains an element that must return the new file name.

    Providing different paths in remotePath and newRemotePath tags, the selected file is moved.

    Examples 
    <ftpRename>
    	<host>
    		<string>host.name</string>
    	</host>
    	<port>
    		<literal value="2121"/>
    	</port>
    	<user>
    		<string>username</string>
    	</user>
    	<password>
    		<string>hrjrj</string>
    	</password>
    	<remotePath>
    		<string>/remote/path/file.tar</string>
    	</remotePath>
    	<newRemotePath>
    		<string>/remote/path/mydata.tar</string>
    	</newRemotePath>
    </ftpRename>

    Renames the file in /remote/path/file.tar to mydata.tar.

    <ftpRename>
    	<host>
    		<string>host.name</string>
    	</host>
    	<port>
    		<literal value="2121"/>
    	</port>
    	<user>
    		<string>username</string>
    	</user>
    	<password>
    		<string>hrjrj</string>
    	</password>
    	<remotePath>
    		<string>/remote/path/file.tar</string>
    	</remotePath>
    	<newRemotePath>
    		<string>/remote/newDirectory/file.tar</string>
    	</newRemotePath>
    </ftpRename>

    Moves the file in /remote/path/file.tar to the directory /remote/newDirectory.

     

    Tag Name updateFtpPermission
    Attributes
    Name Type Use Default Description
    user xsd:string   ascii The user name whose write permissions shall be updated
    permission xsd:boolean required   The new write permission associated to the account
    Description This tag allows the user to update the FTP write permission associated with a user account.
    Examples 
    <updateFtpPermission user="HMA" permission="TRUE"/>
    This tag enables the write permission for the HMA Ftp account.
