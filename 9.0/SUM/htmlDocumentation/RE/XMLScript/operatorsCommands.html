Other Operators and Commands

Tag Name character
Attributes
Name Type Use Default Description
value required The character value.
Description Returns an instance of Character Class. The character value is given by the value attribute. This tag can be used wherever a char is needed.
Examples 
<character value="a" />
Returns the instance of the class Character wrapping the character 'a'.

Tag Name classLiteral
Attributes
Name Type Use Default Description
class xsd:string required The name of the class
Description Returns the object of class Class whose name is given in attribute "class".
Examples 
<classLiteral class="java.io.File"/>
Returns the object of class Class whose name is "java.io.File".

Tag Name command
Attributes
Name Type Use Default Description
asynchronous xsd:boolean false
Flag indicating whether evaluation of the command must wait for the command to complete (default) or not ("asynchronous" set to true).
Description Executes a system command. The command string is given by the enclosed child element. The value of the asynchronous attribute (default false) tells if the tag evaluation must wait for the command to complete (false) or not (true).
Examples 
<command>
	<string>C:\MyCommand parameter1 parameter2</string>
</command>
Executes the command "MyCommand". Evaluation of the tag doesn't wait for completition of command execution. 
<command asynchronous="true">
	<string>C:\MyCommand parameter1 parameter2</string>
</command>
Executes the command "MyCommand". Evaluation of the tag waits for completition of command execution.

Tag Name existsVariable
Attributes
Name Type Use Default Description
name xsd:string required The name of the variable.
Description Returns true if and only if there's an object stored in a variable whose name is given by "name" attribute.
Examples 
<existsVariable name="aVariable"/>
If the variable "aVariable" exists returns "true", else returns "false".

Tag Name field
Attributes
Name Type Use Default Description
fieldName xsd:string required The name of the field storing the object to be returned
Description Returns the object stored in field whose name is given by the "fieldName" attribute of what is specified in the element enclosed in this one. If the enclosed tag is "object", the field is taken from the object returned by the element enclosed in the "object" element. Otherwise the the tag is "class" and the value of the static field of the Class object returned by the element enclosed in the "class" element is returned
Examples 
<field fieldName="anObjectField">
		<object>
			<variable name="anObject"/>
		</object>
</field>
Returns the object stored in the field "anObjectField" of the object ="anObject". 
<field fieldName="obiectField">
		<class>
			<classLiteral class="package.name.ClassName"/>
		</class>
</field>
Returns the object stored in the static field of the class "ClassName" of the package "package.name".

Tag Name generateError
Description Produces an error that stops the XML script execution. The error contains the string given by the enclosed element. This causes the script to be aborted unless the error is caught using the try tag.
Examples 
<if>
	<isNull>
		<variable name="anObject"/>
	</isNull>
	<generateError>
		<string>Script aborted!</string>
	</generateError>
</if>
If the object stored in the variable "anObject" is null, an error stopping the script is produced. The error message is "Script Aborted!". 
<if>
	<isNull>
		<variable name="anObject"/>
	</isNull>
	<try>
		<generateError>
			<string>Script aborted!</string>
		</generateError>
		<ifError errorMessageName="message">
			<print newLine="true">
				<variable name="message"/>
			</print>
		</ifError>
	</try>
</if>
If the variable stored in the variable "anObject" is null, an error (with error message "Script aborted!") is produced. The script is not aborted and the string "Script aborted!" is printed on video.

Tag Name literal
Attributes
Name Type Use Default Description
type TypeType int The primitive type to be wrapped
value xsd:string required The value of the primitive type to be wrapped
Description Returns an instance of a wrapper class (Byte, Short, Integer etc.) whose type is given by the "type" attribute and whose value is given by the "value" attribute. This tag can be used wherever a primitive type is needed.
Examples 
<literal value="3"/>
Returns an integer with value 3. 
<literal type="boolean" value="true"/>
Returns a boolean with value true.

Tag Name log
Attributes
Name Type Use Default Description
level com:LogLevelType INFO
The log level. Five values are possible: FATAL, WARN, ERROR, INFO (default), DEBUG, ALL
Description Produces a log in the service log. The logged string is given by the enclosed element. The level attribute selectes the log level (available levels in verbosity order: FATAL, WARN, ERROR, INFO, DEBUG, ALL)
Examples 
<log>
	<stringCat>
		<string>File Size: </string>
		<variable name="fileSize"/>
	</stringCat>
</log>
Produces in the service log file a log message at level "INFO", containing the string resulting by the concatenation of "File Size: " and the string stored in the variable "fileSize".

Tag Name methodInvocation
Attributes
Name Type Use Default Description
methodName xsd:string required The name of the method to be invoked
Description Invokes a method of an object or a static method of a class and returns the value returned by that method. The name of the method is taken from the "methodName" attribute. If the first enclosed element has tag "object", the method is called on the object returned by the element enclosed in the "object" element, otherwise it has tag "class" and the static method of the Class object returned by the element enclosed in the "class" element is invoked. The "object" and the "class" elements can be followed by a sequence of elements tagged "parameter", used to distinguish between overloaded methods and to retrieve the parameters to be passed to the method. The elements "parameter" have two enclosed elements: the first one returns the object of class Class the parameter belongs to, the second one returns the value of the parameter.
Examples 
<methodInvocation methodName="readInt">
	<object>
		<variable name="in"/>
	</object>
</methodInvocation>
Returns the value returned by the invocation of the "readInt" method on the the object stored in the "in" variable. 
<methodInvocation methodName="getAOILabel">
	<class>
		<classLiteral class="AOIFetcher"/>
	</class>
	<parameter>
		<classLiteral class="java.lang.String"/>
		<variable name="label"/>
	</parameter>
	<parameter>
		<classLiteral class="java.lang.String"/>
		<variable name="coordinates"/>
	</parameter>
</methodInvocation>

Returns the value returned by the invocation of the static method "getOAILabel" of the class "AOIFetcher" with the following parameters: the first one is a String whose value is stored in the variable "label", the second one is another string whose value is stored in the variable "coordinates".

Invoked methods must belong to classes contained in the TOOLBOX CLASSPATH, which includes the Tomcat CLASSPATH, too. To add class libraries, copy the ".jar" file containing the library under <TOMCAT_HOME>/webapps/TOOLBOX/WEB-INF/lib/ and reload the TOOLBOX according to the following procedure:

  • point the browser to "http://<TOMCAT_HOST>[:<TOMCAT_PORT>]/manager/reload?path=/TOOLBOX"
  • fill the possibly prompt window with username and password chosen during the Tomcat installation.
  • Log in again the TOOLBOX.
    •

    In the example above, a ".jar" file (named "AOIFetcher.jar") containing the "AOIFetcher" class has been stored under <TOMCAT_HOME>/webapps/TOOLBOX/WEB-INF/lib/. Remember that class names must be fully qualified (i.e. must include the package name). In this case AOIFetcher, belongs to the anonymous package.

    Tag Name newInternalObject
    Attributes
    Name Type Use Default Description
    internalClass xsd:string required The name of the internal class the object to be created belongs to.
    Description Creates a new object, instance of an internal class, and returns it. The name of the internal class is given by the "internalClass" attribute. The first enclosed element must return the object that will contain that instance to be created. The first element can be followed by a sequence of elements tagged "parameter", used to distinguish between overloaded constructors and to retrieve the parameters to be passed to the constructor. The elements "parameter" have two enclosed elements: the first one returns the object of class Class the parameter belongs to, the second one returns the value of the parameter.
    Examples 
    <newInternalObject internalClass="InternalClassName">
    	<variable name="OuterClass"/>
    	<parameter>
    		<string>java.lang.Integer</string>
    		<variable name="IntergerValue"/>
    	</parameter>
    </newInternalObject>
    Creates an instance on the internal class "InternalClassName", whose outer class is ""OuterClass". The Integer object stored in the variable "IntergerValue" is passed to the constructor.

    Tag Name newObject
    Description Creates a new object and returns it. The first enclosed element must return the class of the object to be created. The first element can be followed by a sequence of elements tagged "parameter", used to distinguish between overloaded constructors and to retrieve the parameters to be passed to the constructor. The elements "parameter" have two enclosed elements: the first one returns the object of class Class the parameter belongs to, the second one returns the value of the parameter.
    Examples 
    <newObject>
    	<classLiteral class="java.io.File"/>
    	<parameter>
    		<classLiteral class="java.lang.String"/>
    		<variable name="pathProduct"/>
    	</parameter>
    </newObject>
    Returns ao object File whose path is stored in the variable "pathProduct".

    Tag Name randomInt
    Attributes
    Name Type Use Default Description
    maxExclusive xsd:positiveInteger optional The maximum exclusive returnable value.
    Description Produces a random integer. If the (optional) "maxExclusive" attribute is present, the returned value is comprised between 0 (inclusive) and the attribute value (exclusive). Otherwise the returned value is comprised between -2^31 (inclusive) and (2^31) - 1 (inclusive).
    Examples 
    <randomInt maxExclusive="1000"/>
    Returns a random integer comprised between 0 (inclusive) and 1000 (exclusive). 
    <randomInt/>
    Returns a random integer comprised between -2^31 (inclusive) and (2^31) - 1 (inclusive).

    Tag Name sequence
    Description It is just a tag container. It can be empty or contain any number of child tags of any kind. Useful when a single element is needed, but a complex operation has to be performed. The value returned by this tag is the value returned by the last enclosed tag, or null if it is empty.
    Examples 
    <sequence/>
    Returns null. 
    <sequence>
    	<setVariable name="x">
    		<literal value="5"/>
    	</setVariable>
    	<!-- Other stuff not modifying variable x -->
    	<variable name="x"/>
    </sequence>
    Returns 5 
    <not>
    	<sequence>
    		<!-- Any complex stuff setting boolean variable x to true -->
    		<variable name="x"/>
    	</sequence>
    </not>
    Returns false.

    Tag Name sleep
    Attributes
    Name Type Use Default Description
    amount xsd:positiveInteger required The amount of sleep time of the engine
    Description Causes the engine to stop execution for a while. The sleep time is given by the amount attribute, expressed in milliseconds.
    Examples 
    <sleep amount="30000"/>
    The engine stops execution for 30 seconds.

    Tag Name timer
    Attributes
    Name Type Use Default Description
    delay com:ToolboxTime required The amount of time elapsed which the Toolbox must execute a script. This value is expressed by a non-negative number followed by a letter (s=second m=minute h=hour d=day w=week). It is possible to set this attribute through the $variable syntax
    Description this tag tells the Toolbox to execute a script after a given time. The execution of the timed script is not part of the launching script. The delay is given by the "delay" attribute, the value has the same format as the "duration" of ftpAccount. The script to be executed is the enclosed element (this is not the value returned by the evaluation of the element, but the DOM sub-tree itself!).
    Examples 
    <timer delay="8h">
    	<fileDelete>
    		<string>C:\MyFile.txt</string>
    	</fileDelete>
    </timer>
    8 hours after the execution of the "timer" tag the file MyFile is deleted.

    Tag Name variable
    Attributes
    Name Type Use Default Description
    name xsd:string required The name of the variable storing the object to be returned
    Description Returns the object stored in variable whose name is given by "name" attribute. If nothing has been stored with that name, it returns null.
    Examples 
    <variable name="variableName"/>
    Returns the object stored in the variable "variableName".
