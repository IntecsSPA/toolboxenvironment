XML Scripting

Tag Name call
Description This tags execute an XML script previously stored in a variable. The script could have been stored by the tag loadProcedure, or storeProcecure. The structure and the behaviour of the tag is as the "execute" tag but the "name" attribute referes to a variable name instead of the path of a file. The tag "call" can have a sequence of child tags "argument" specifiyng the parameters used by the XML script. Each tag "argument" has a mandatory attribute "name" containing the name of the parameter and an enclosed element returning the value of the parameter.
Examples 
<storeProcedure name="myProcedure">
	<sequence>
		<!-- My frequently used procedure -->
	</sequence>
</storeProcedure>
	.
	.
	.
<call procedure="myProcedure">
	<argument name="MyPar1">
		<string>MyValue1</string>
	</argument>
	<argument name="MyPar2">
		<literal value="10"/>
	</argument>
</call>
The tag "call" sets MyPar1 variable with "MyValue1" string and MyPar2 variable with integer 10, then executes the script stored in variable "myProcedure" and returns the result of the XML script.

Tag Name execute
Description This tags loads an XMl script from a file on disk and executes it. e file name is given by the "procedure" element. The enclosed "argument" elements are completely equivalent to the setVariable element. The variables set in this way are visible not only to the script that is going to be executed, but also to the rest of the script. This tag returns the value returned by the root tag of the loaded XMl script.
Examples 
<execute>
	<procedure>
		<string>MyScriptDir\MyProcedure.xml<string>
	</procedure>
	<argument name="MyPar1">
		<string>MyValue1</string>
	</argument>
	<argument name="MyPar2">
		<literal value="10"/>
	</argument>
</execute>
Sets MyPar1 variable with "MyValue1" string, MyPar2 variable with integer 10, loads MyScriptDir\MyProcedure.xml file, and executes it. This tag should be used when a common procedure is stored on a separated file, but this procedure is not executed many times during the script, because file loading is performed each time
   

Tag Name loadProcedure
Attributes
Name Type Use Default Description
name xsd:string required The name of the file containing the XMLscript.
Description This tag loads an XML script from a file (whose name is represented by the string returned by the enclosed element) on disk and stores the script in a variable(named as attribute "name"). This tag returns the loaded script.
Examples 
<loadProcedure name="myProcedure">
	<string>C:\XML\myXMLScript.xml</string>
</loadProcedure>
Loads the procedure stored in the file "C:\XML\myXMLScript.xml" and stores it in the variable "myProcedure". The script can be later executed using the tag "call".

Tag Name storeProcedure
Attributes
Name Type Use Default Description
name xsd:string required The name of the variable storing the script
Description This tags stores the script defined by the enclosed element in a variable named as attribute "name". This tag returns the stored script.
Examples 
<storeProcedure name="myProcedure">
	<sequence>
		<!-- My frequently used procedure -->
	</sequence>
</storeProcedure>
Stores the procedure whose root tag is "sequence" in the variable "myProcedure". The script can be later executed using the tag "call".
