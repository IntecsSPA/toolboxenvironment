Error Messages

Below the list of the error messages that can be sent to the SOAP client/server (SSE portal) is reported:

Synchronous operations

Message Meaning
"Invalid request: <reason>"

The request sent from the SSE Portal is not valid against the schema set installed on the TOOLBOX. <reason> is a string explaining why the request is not valid.
"Error during script execution: <reason>" The execution of the script failed. <reason> is a string (returned by the Java Run Time Environment) telling the cause of the failure.
"Invalid response: <reason>" The response generated by the script is not valid against the schema set installed on the TOOLBOX. <reason> is a string explaining why the request is not valid.

Asynchronous operations

Message Meaning
"Impossible to extract SOAP Header: <reason>"

the TOOLBOX was unable to extract the SOAP Header from the SOAP message. <reason> is a string telling the cause of the failure.
"Impossible to build request pack: <reason>" the TOOLBOX was unable to extract some information from the SOAP Message or the push url contained in the SOAP HEader is not allowed according to the service configuration. <reason> can be one of the following:
  • "Error getting pushURL: <reason_for_error>"
  • "Wrong host: <host>"
  • "Error getting messageId: <reason_for_error>"
  • "Error getting orderId: <reason_for_error>"
"Impossible to create new request handler: messageId in use: <messageId>" A request having the same messageId is already in the service status
"Invalid request: <reason>" The request sent from the SSE Portal is not valid against the schema set installed on the TOOLBOX. <reason> is a string explaining why the request is not valid.
"Request in wrong status: <statusString>" The TOOLBOX found the request in one of the error status (a status different from "pending") when tried to perform the second script, <statusString> indicates the status.

"Error during script execution: <reason>"		 The TOOLBOX failed to executes the second or the third script (if failed to execute the first one, the error message is "Request in wrong status: aborted"). <reason> is a string telling the cause of the failure.
"Invalid response: <reason>" The response generated by the script is not valid against the schema set installed on the TOOLBOX. <reason> is a string explaining why the request is not valid.
