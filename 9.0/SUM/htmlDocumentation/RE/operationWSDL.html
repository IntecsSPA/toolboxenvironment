WSDL

The TOOLBOX support the automatic WSDL generation. When a new service is installed and the first operation has been added, the TOOLBOX publishes the WSDL according to this name convention

http://<HOST>[:<PORT]>/TOOLBOX/WSDL/<SERVICE_NAME>/<SERVICE_NAME>.wsdl
where <HOST> can be the host address of Tomcat or Apache (depending on the hardware configuration) and <PORT> can be the port number where Tomcat or Apache are listening on (depending on the hardware configuration).

The WSDL file will have the structure specified in the SSE ICD , even for services whose (TOOLBOX) SOAP interfaces will not support the ICD.

When a SSE operation is published, the WSDL excerpt related to the new operation ia automatically added to the WSDL file that describe the service interface.

For generic operations (operation which are not compliant with the SSE ICD) more details are needed. Thus when the operation is published it is not added to the WSDL file. To add the new operation to the WSDL file follow the "Adding an operation to the WSDL" section.
