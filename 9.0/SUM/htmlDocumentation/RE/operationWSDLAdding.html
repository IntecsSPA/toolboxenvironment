Adding an operation to the WSDL

For all operation whose interface has automatically created from the interface repository, the WSDL parameters a pre-defined. Thus this procedure is not needed.

To add a generic operation to the WSDL file follow these steps.

Display the operations list then click on the "info" link. The Browser will display a page reporting the information needed to add the operation to the WSDL file.

Click on the "Configure >>" button. the browser will display two different pages depending on the type of operation you have selected.

Fill the form as reported below.

Synchronous Operations

  • Input Message
    • Name: name of the element (included in the service schema) representing the root of the payload of the SOAP requests addressed to the service.
    • Namespace URI: Namespace URI of the service schema element specified by the "Name" field.
  • Output Message
    • Name: name of the element (included in the service schema) representing the root of the payload of the SOAP response returned by the service.
    • Namespace URI: Namespace URI of the service schema element specified by the "Name" field.

Asynchronous operations

  • Input Message
    • Name: name of the element (included in the service schema) representing the root of the payload of the SOAP requests addressed to the service.
    • Namespace URI: Namespace URI of the service schema element specified by the "Name" field.
  • Output Message
    • Name: name of the element (included in the service schema) representing the root of the payload of the SOAP response returned by the service.
    • Namespace URI: Namespace URI of the service schema element specified by the "Name" field.
  • Push Message
    • Name: name of the element (included in the service schema) representing the root of the payload of the SOAP messages pushed by the TOOLBOX to the SOAP server (e.g. SSE).
    • Namespace URI: Namespace URI of the service schema element specified by the "Name" field.
    • SOAP Action: value of the SOAPAction header to set in the HTTP message that carries the SOAP message pushed to the SOAP server (e.g. SSE).
  • Push Response
    • Name: name of the element (included in the service schema) representing the root of the payload of the SOAP messages returned by the SOAP server (e.g. SSE) as responses to the messages pushed by the TOOLBOX.
    • Namespace URI: Namespace URI of the service schema element specified by the "Name" field.

Click on the "Configure" button to update the WSDL information.
