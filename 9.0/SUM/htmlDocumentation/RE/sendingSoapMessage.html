Sending a SOAP message

To test a service directly from the TOOLBOX Web Application, click on the "Test center" link on the toolbar, then click on the Send a "Toolbox application client" link. A new browser page is opened, displaying the test center. The tools implements a desktop like environment in which 3 different kind of testing tools can be accessed from the bottom bar.

Toolbox services client

Clicking on this icon, a dialog is opened. This dialog allows the user to speficy the URL of the Toolbox where  the service to test is currently running.  When inserted, click on the button "Retrieve services". Through this the list of services is retrieved and the section on the bottom of the dialog filled. Then select the service and the operation to test. When done, click on the button "Create Request".
A new dialog page is shown, where the content of the test message can be specified.
Select one of the following content types:
  • Synchronous Payload: select this radio box to provide the XML content to be included in the SOAP message in order to send a synchronous request.
  • Asynchronous Payload: select this radio box to provide the XML content to be included in the SOAP message in order to send a asynchronous request.
  • SOAP Message:select this radio box to provide directly the SOAP message to be sent.
Click on "Browse" and select the XML file to be uploaded. Click on the button "Load message" to load the specified XML message in the editing area. If a change is necessary, simply edit the file directly.
To send the message to the service, click on the button "Send request". The message is sent and the response is displayed in a new dialog page for investigation.

See the Push Server page to know how to use the push server to test asynchronous operations.
