Configuring the TOOLBOX

To configure the TOOLBOX you have to log into the TOOLBOX.

The configuration page is automatically displayed at the first login after the installation. In this case the following two steps have to be skipped.

To configure the TOOLBOX click on the "TOOLBOX configuration" link on the toolbar. The browser should display

click on the "Change the configuration" section link. The browser should display

Ignore the possible values in the various fields: they have to be customized!

The main table contains three sets of parameters, depending on the type of settings:

Functional Settings - MANDATORY
  • Queue incoming service requests – indicates the wished requests queuing level. By setting this parameter to "true", all the requests to the Toolbox will be queued (global queuing), avoiding concurrence between the executions of the send scripts of all the services.
  • Check for new TOOLBOX version at login time – indicates if the TOOLBOX has to look for new version of the tool when the Administrator perform the login procedure. If this parameter is checked and a new version of the TOOLBOX is available, a page with some information on the new version and a link for the new package download will be displayed.
  • Check for new SSE Schema version at service configuration/creation time – indicates if the TOOLBOX has to look for new version of the SSE ICD when the administrator configure a service which is using the SSE Schemas. If this parameter is checked and a new version of the Schema is available a warning icon is displayed on the top of the page.
FTP Settings
  • FTP Administrator Directory – indicates the path to the directory of the administrator of the FTP server embedded in the TOOLBOX.
  • FTP Administrator Password - indicates password of the administrator of the FTP server embedded in the TOOLBOX.
  • FTP Port Number - indicates the port number the FTP server embedded in the TOOLBOX receives requests from.
  • FTP Passive Mode Port Numbers - indicates the allowed ports number for the PASV mode. Note: If the server is behind firewall, these are firewall holes.
  • FTP Server Host - Behind firewall, the FTP server needs a real inet address that from ISP.
Log Settings (MANDATORY)
  • Logging Level – indicates the level of verbosity of the log files. Seven values are possible:
    • ALL – this level has the lowest possible rank and is intended to turn on all logging
    • DEBUG - this level designates fine-grained informational events that are most useful to debug the application.
    • INFO - this level designates informational messages that highlight the progress of the application at coarse-grained level.
    • WARN - This level designates potentially harmful situations.
    • ERROR - This level designates error events that might still allow the application to continue running.
    • FATAL - This level designates very severe error events that will presumably lead the application to abort.
    • OFF - The OFF has the highest possible rank and is intended to turn off logging.
  • Log Directory – indicates the path to the directory holding the log files and possibly the SOAP messages.
  • Log File Size – indicates the size of the log file. The log functionality uses a rolling policy. When the file size is reached a new log file is automatically created. The old log files are renamed using an index. Nine backup files are stored on disk for each service. Thus the maximum disk space required for each service for logging purposes is 10*<Log File Size>.
Tomcat installation (depending on hardware configuration)
  • Port number: It indicates the port number Tomcat is listening on and it is used by the TOOLBOX to automatically generate the WSDL files related to the deployed services
  • SSL Port Number: It indicates the port number Tomcat is listening on for secure communication and it is used by the TOOLBOX to automatically generate the WSDL files related to the deployed services.

Apache installation (depending on hardware configuration)

If you need to let TOOLBOX answer to incoming requests to port 80 you need to install Apache with proxy extension. If properly set, all incoming requests (web page access, SOAP messages etc.) are transparently forwarded to Toolbox. For a detailed description on how setup Apache to provide a proxy, please read here.

These parameters have to be filled only if Apache is used as proxy server.

  • Host name – this field is to be filled only if Tomcat runs behind Apache. The Apache host name is used by the TOOLBOX to automatically generate the WSDL files related to the deployed services.
  • Port Number - this field is to be filled only if Tomcat runs behind Apache and it is used by the TOOLBOX to automatically generate the WSDL files related to the deployed services.
Proxy settings (depending on local network configuration)

These parameters have to be filled only if the local network uses a proxy to route outgoing HTTP messages.

  • Host name – The host name the proxy is running on.
  • Port Number - The port number (of the proxy host) the proxy is listening on.
Contact information

This information will be used in the email sent to the Service Support Environment Operations Team each time a request arrived to the TOOLBOX fails. Moreover it is included in the info page deployed when a new service is created. This page is referenced in the WSIL file.

  • Company name: The name of the company where the TOOLBOX is installed in.
  • Company reference person : the name of the reference person which can be contacted in case of problems during the TOOLBOX processing.
  • Email address : The e-mail address of the Company reference person. This account must exist on the SMTP server specified in the field SMTP server.
Error report to the Service Support Environment Operations Team

The service provider can optionally choose to send an e-mail to the Service Support Environment Operations Team each time a request arrived to the TOOLBOX fails. The Service Support Environment Operations Team e-mail address is known to the TOOLBOX and the service provider is not required to specify it. Instead, the service provider can optionally specify other recipients for the error report e-mail.

  • Error report to the Service Support Environment Operations Team: If set to true, an e-mail is sent to the Service Support Environment Operations Team each time a request arrived to the TOOLBOX fails.
  • Mail error report to other recipients: If set to true, an e-mail is sent to the other recipients each time a request arrived to the TOOLBOX fails.
  • Other recipients : The e- mail addresses of the other recipients, apart of the Service Support Environment Operations Team, of the error report e-mail (the list must be blank space-separated).
  • SMTP server : The host name of the SMTP server to be used to send the error report e-mail. The account specified in the field Sender must exist on this server.
ebRR configuration parameters

This section allows the administrator to configure ebRR specific parameter.

  • Repository Home Directory: This configuration parameter sets the root directory in which ebRR stores all documents during the harvest operation. This directory is also used during the GetRepositoryItem to retrieve harvested documents.
Security
This section allows the administrator to configure a keystore at Toolbol level. These settings will be used when security enforcement is enabled on a service.  In these case the keystore should contain the key of the Identity Provider so that the SAML token, taken from the incoming message, can be decrypted and, in case of proxy services, re-encrypted with the IDP public key. For further information about securing a service, please refer to the specific section (here)
  • Toolbox level keystore: This configuration parameter sets the keystore file to be used. Click on browse to upload a new keystore file.
  • Keystore password: Password to be used for accessing the key. All keys are tested until a match or a failure is found.


After having filled the fields above, click on the "Configure" button. The browser should display

Now the TOOLBOX has to be reloaded in order to apply all settings. Restart Tomcat for a full configuration reload.
