Configuration of metadata extraction shell scripts

The ARMS can be configured in order to be capable of extracting metadata from downloaded data. The solution provided by the ARMS will allow the administrator to associate a shell script file to a data type. Each time a data is downloaded and the client hasn't specified a metadata document, the shell script is executed. If metadata is produced successfully it is harvested into the ebRIM catalogue specified in the store operation input message (see here for a description).In order to add a shell script, the administrator has to execute the following steps:
  • Copy the shell script under <APACHE_TOMCAT>/webapps/ArchivingServer/WEB-INF/metadataExtractors
  • Open the file <APACHE_TOMCAT>/webapps/ArchivingServer/WEB-INF/classes/metadataExtractionProcessings.properties
  • Edit the file. Find the key matching the data type and put the file name of the shell script as value. If the key is not available, add a new one
  • Save the properties file
The system will automatically fetch the change without need of reloading the application or Tomcat. The ARMS servlet will identify the data type through the DataType field available in the input message. Depending on the interface used, the DataType field may differ. Please refer to the section Interfaces for a detailed description.
