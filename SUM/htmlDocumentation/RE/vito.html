File exchange based service

In the following we report a detailed description of a file exchange based service.

This is a real service provided by VITO and it has been successfully integrated in the in the SSE framework. This description reports the steps needed to expose the VITO NPP service as Web Service compliant with the SSE framework (script development, service deployment and testing). The definition of the Schema that defines the service, the development of the style sheet needed to integrate the service in the SSE Portal and the integration itself of the VITO NPP Web Service in the SSE Portal is out of the scope of this tutorial. On this subject, please see the SSE ICD.

The Context

Net Primary Productivity or NPP products represent repeatable estimates of the net flux of carbon between the atmosphere and terrestrial vegetation. It quantifies these carbon fluxes by combining a simplified carbon exchange model with satellite observations (SPOT-VEGETATION S10 (SPOT VGT S10) images).

  • This product is available on 10 predefined regions of interest (ROI): Africa, North America, Central America, South America, S-Asia, W-Asia, SE-Asia, Asian Islands, Australasia and Europe
  • period: April 1998 - present
  • frequency: every 10 days
  • type: raster
  • resolution: 1 km x 1 km
  • format: HDF

Each time a new product is ready it is stored in a directory identified according to a naming convention which is based on the parameter which define the product itself (e.g. ROI, time range etc.).

The TOOLBOX has been used to expose this directory structure as a Web Service.

When a new sendOrder request arrives, the TOOLBOX looks for the specific product in the Products Directory. The file name and location is built using the request parameters. If the file is present in the specified directory and the user quota has not been exceeded, the TOOLBOX creates a link pointing to the product in the delivery directory and create an FTP account. Then it sends back a successful message containing the product size, the available quota (a single user has a pre-defined amount of virtual disk space dedicated to the product download), ftp address ftp username and password. Otherwise the TOOLBOX sends back an error message containing the sentence “Product not available” (if the product is not included in the products directory) or “Error: Quota Exceeded.” (if the product size exceeds the available user quota). The End User can download the ordered product using the information (ftp URL, username and password) included in the response message.

The product file name for the NPP products is coded using the following convention

V1KRNS10__YYYYMMDD_NEP___ROI.ZIP

with:

  • YYYY: year
  • MM: month
  • DD: day
  • ROI: Region of interest.

All the ordered products will be stored in the FTP root directory coded using the following convention /FTP_MAIN_ROOT/userId with:

  • FTP_MAIN_ROOT: is the main FTP directory
  • userId: is the user identifier extracted from the order message

The Script development

We suppose that:

  • The Linux OS is running on the target machine.
  • A "toolbox" user (home directory /home/toolbox/) has been created.
  • The Schema needed to validate the incoming and outgoing messages has been created according to the SSE ICD.
  • All the files needed to integrate the service are located in the /home/toolbox/VITO/NPP/ directory.

The full version of the script is reported in Listing 1. Hereafter we report a detailed description of the main tags.

Setting the User Quota.

The User Quota is the amount of disk space reserved for each user. The User Quota is set (<setVariable> tag) to 5GB (variable maxUserQuota) by the following script excerpt

                              <setVariable name="maxUserQuota">
                                    <literal value="5248"/>
                              </setVariable>

Extracting the date

The file location is built using, among other parameters, the date included in the query parameters. The format of the date used in the file system is YYYYMMDD. The format of the date included in the SOAP message is YYYY-MM-DD. Thus the following script excerpt is used to build the correct data format using xslt (extractDate.xsl).

                              <setVariable name="realDate">
                                    <xslt>
                                           <xmlRequest/>
                                           <string>/home/toolbox/VITO/NPP/extractDate.xsl</string>
                                    </xslt>
                              </setVariable>

Extracting the coordinates and the label

The AOI in the input SOAP message can be described in two different way: via a label or via a bounding box. Both elements can be present at the same time. These parameters are extracted using the <xPath> tag and stored into two variables

                              <setVariable name="coordinates">
                                    <xPath xmlns:mass="http://www.esa.int/mass" xmlns:aoi="http://www.esa.int/xml/schemas/mass/aoifeatures"
                                       xmlns:gml="http://www.opengis.net/gml">
                                           <xmlRequest/>
                                           <string>/mass:sendOrderInputMsg/mass:sendOrderInput/
                                                        aoi:areaOfInterest/gml:boundedBy/gml:Box/gml:coordinates</string>
                                    </xPath>
                              </setVariable>
                              <setVariable name="label">
                                    <xPath xmlns:mass="http://www.esa.int/mass" xmlns:aoi="http://www.esa.int/xml/schemas/mass/aoifeatures"
                                      xmlns:gml="http://www.opengis.net/gml">
                                           <xmlRequest/>
                                           <string>/mass:sendOrderInputMsg/mass:sendOrderInput/
                                                       aoi:areaOfInterest/gml:featureMember/aoi:Feature/aoi:Label</string>
                                    </xPath>
                              </setVariable>

Creating the file name

The AOI in the input SOAP message can be described in two different way: by means of a label or by means of a bounding box. If the label is provided, no additional computation is needed. If the AOI is provided by means of a bounding box, the coordinates have to be converted into a label according to a table provided by VITO. This operation is performed via a JAVA method (getAOILabel(java.lang.String label ,java.lang.String coordinates) of the class AOIFetcher) which is then executed via the <methodInvocation> tag. The result of this method is then concatenated (with the <stringCat> tag ) with the realDate variable and some other fixed strings to build the name of the file to search for. The result of the concatenation is then stored in the shortNameFile variable. 

                              <setVariable name="shortNameFile">
                                    <stringCat>
                                           <string>V1KRNS10__</string>
                                           <variable name="realDate"/>
                                           <string>_NPP___</string>
                                           <methodInvocation methodName="getAOILabel">
                                                  <class>
                                                        <classLiteral class="AOIFetcher"/>
                                                  </class>
                                                  <parameter>
                                                        <classLiteral class="java.lang.String"/>
                                                        <variable name="label"/>
                                                  </parameter>
                                                  <parameter>
                                                        <classLiteral class="java.lang.String"/>
                                                        <variable name="coordinates"/>
                                                  </parameter>
                                           </methodInvocation>
                                    </stringCat>
                              </setVariable>

Creating the complete path

The file name is then concatenated with a fixed string (/data/VGT1/S10/) and with the date to build the absolute path of the data file. The result of this concatenation is then stored in the fileName variable.

                              <setVariable name="fileName">
                                    <stringCat>
                                           <string>/data/VGT1/S10/</string>
                                           <variable name="realDate"/>
                                           <string>/</string>
                                           <variable name="shortNameFile"/>
                                           <string>.ZIP</string>
                                    </stringCat>
                              </setVariable>

Checking if the file is available

The script checks <if> the product is available

      <if>
             <fileExists>
                    <variable name="fileName"/>
             </fileExists>
             <sequence>
....
             </sequence>

If not, an error message compliant with the SSE ICD is sent back. The error message is built using the <xml> tag.

                                           <xml attributePrefix="attribute$" textTag="evaluate">
                                                  <mass:returnOrderResultInputMsg xmlns:mass="http://www.esa.int/mass">
                                                        <mass:commonInput>
                                                               <mass:orderId><evaluate name="orderID"/></mass:orderId>
                                                        </mass:commonInput>
                                                        <mass:getOrderOutput>
                                                               <mass:statusInfo>
                                                                      <mass:statusId>300</mass:statusId>
                                                                      <mass:statusMsg>Product Not Found</mass:statusMsg>
                                                               </mass:statusInfo>
                                                        </mass:getOrderOutput>
                                                  </mass:returnOrderResultInputMsg>
                                           </xml>

Creating the user directory

The service creates a directory for each user. This directory is used to store the products (or better the link to the products) ordered by the user. The absolute path of this directory is created concatenating a fixed string (/home/toolbox/data/NPP/users/) with the user name (mass:userId) extracted (using the <xPath> tag) from the SOAP request payload (<xmlRequest> tag)

                              <setVariable name="userName">
                                    <xPath xmlns:mass="http://www.esa.int/mass">
                                           <xmlRequest/>
                                           <string>/mass:sendOrderInputMsg/mass:sendOrderInput/mass:userId</string>
                                    </xPath>
                              </setVariable>
                              <setVariable name="userDirectory">
                                    <stringCat>
                                           <string>/home/toolbox/data/NPP/users/</string>
                                           <variable name="userName"/>
                                    </stringCat>
                              </setVariable>

The directory is then created using the <mkdir> tag

       <mkdir>
             <variable name="userDirectory"/>
       </mkdir>

Calculating the User Quota

The User Quota is the total size of the products ordered by a single user. It is calculated by summing the size of the files stored in the user directory. The total size is calculated executing (<execute> tag) an external script (/home/toolbox/VITO/scripts/getQuota.xml). The sum is then divided by 1024 and stored in the userQuota variable 

       <setVariable name="userQuota">
             <itoa>
                    <divide>
                           <execute>
                                 <procedure>
                                        <string>/home/toolbox/VITO/scripts/getQuota.xml</string>
                                 </procedure>
                                 <argument name="downloadDir">
                                        <newObject>
                                               <classLiteral class="java.io.File"/>
                                               <parameter>
                                                     <classLiteral class="java.lang.String"/>
                                                     <string>/home/toolbox/data/NPP/users</string>
                                               </parameter>
                                        </newObject>
                                 </argument>
                                                        <argument name="name">
                                        <variable name="userName"/>
                                 </argument>
                           </execute>
                           <literal value="1024"/>
                    </divide>
             </itoa>
       </setVariable>

Calculating the data file size

The size of the ordered product is calculated invoking (<methodInvocation>) the "length()" method of the "java.io.File" Java class and dividing the result by 1024. The result of this operation is stored in the fileSize variable

       <setVariable name="fileSize">
             <itoa>
                    <divide>
                           <methodInvocation methodName="length">
                                                        <object>
                                        <newObject>
                                               <classLiteral class="java.io.File"/>
                                               <parameter>
                                                     <classLiteral class="java.lang.String"/>
                                                     <variable name="fileName"/>
                                               </parameter>
                                        </newObject>
                                 </object>
                           </methodInvocation>
                           <literal value="1024"/>
                    </divide>
             </itoa>
       </setVariable>

Calculating the available quota

The available quota is calculated subtracting (<minus>) the userQuota and the fileSize to the maxUserQuota.

       <setVariable name="availableQuota">
             <itoa>
                    <minus>
                           <atoi>
                                 <variable name="maxUserQuota"/>
                           </atoi>
                           <atoi>
                                 <variable name="userQuota"/>
                           </atoi>
                           <atoi>
                                  <variable name="fileSize"/>
                           </atoi>
                    </minus>
             </itoa>
       </setVariable>

Checking if the quota has been exceeded

<If> the available quota is greater or equal to zero the script goes on.

                    <if>
                           <gte>
                                 <atoi>
                                        <variable name="availableQuota"/>
                                 </atoi>
                                 <literal value="0"/>
                           </gte>
                           <sequence>
....
                           </sequence>

Otherwise, an error message compliant with the SSE ICD is sent back (error message "Quota exceeded"). The error message is built using the <xml> tag.

                                                 <xml attributePrefix="attribute$" textTag="evaluate">
                                                        <mass:returnOrderResultInputMsg xmlns:mass="http://www.esa.int/mass">
                                                               <mass:commonInput>
                                                                      <mass:orderId><evaluate name="orderID"/></mass:orderId>
                                                               </mass:commonInput>
                                                               <mass:getOrderOutput>
                                                                      <mass:statusInfo>
                                                                            <mass:statusId>200</mass:statusId>
                                                                            <mass:statusMsg>Error: Quota Exceeded.</mass:statusMsg>
                                                                      </mass:statusInfo>
                                                               </mass:getOrderOutput>
                                                        </mass:returnOrderResultInputMsg>
                                                  </xml>

Creating the directory for the product download

The script creates the directory that will be used to store the product to download.

                                 <setVariable name="pathProduct">
                                        <stringCat>
                                               <string>/home/toolbox/data/NPP/users/</string>
                                               <variable name="userName"/>
                                               <string>/</string>
                                               <variable name="shortNameFile"/>
                                        </stringCat>
                                 </setVariable>
                                  <mkdir>
                                              <variable name="pathProduct"/>
                                  </mkdir>

Creating the link to the the product to download

The scripts creates (<command> tag) the link to the product that will be accessed via FTP. The linkFile variable will be used in the following to delete the (link to the) product after 5 days.

                                                     <setVariable name="linkFile">
                                                               <stringCat>
                                                                      <variable name="pathProduct"/>
                                                                      <string>/</string>
                                                                      <variable name="shortNameFile"/>
                                                                      <string>.ZIP</string>
                                                               </stringCat>
                                 </setVariable>
                                 <command>
                                        <stringCat>
                                               <string>ln -s </string>
                                               <variable name="fileName"/>
                                               <string> </string>
                                               <variable name="linkFile"/>
                                        </stringCat>
                                 </command>

Defining some variables

The script extracts the orderId from the request message and stores it in the orderID variable

                                 <setVariable name="orderID">
                                        <xPath xmlns:mass="http://www.esa.int/mass">
                                               <xmlRequest/>
                                                                      <string>mass:sendOrderInputMsg/mass:commonInput/mass:orderId</string>
                                        </xPath>
                                 </setVariable>

creates the FTP url to be included in the response message concatenating the localHost tag with the shortName variable and some fixed strings and stores it in the orderResultUrl variable

                                 <setVariable name="orderResultURL">
                                        <stringCat>
                                               <string>ftp://</string>
                                               <localHost/>
                                               <string>:21/</string>
                                               <variable name="shortNameFile"/>
                                               <string>.ZIP</string>
                                        </stringCat>
                                 </setVariable>

sets the timeservice variable to 5

                                 <setVariable name="timeService">
                                        <string>5</string>
                                 </setVariable>

creates a new user account string concatenating the userName variable with a random string (randomString tag) and stores it in the userAccount variable

                                 <setVariable name="userAccount">
                                        <stringCat>
                                               <variable name="userName"/>
                                               <randomString length="3"/>
                                        </stringCat>
                                 </setVariable>

creates an 8 character random password (randomString tag) and stores it in the password variable

                                 <setVariable name="password">
                                        <randomString length="8"/>
                                 </setVariable>

Creating the FTP account

The script creates the ftp account by means of the <ftpAccount> tag (duration 5 days) using the variable defined above

                                 <ftpAccount duration="5d">
                                        <user>
                                               <variable name="userAccount"/>
                                        </user>
                                        <password>
                                               <variable name="password"/>
                                        </password>
                                        <rootDir>
                                               <variable name="pathProduct"/>
                                        </rootDir>
                                 </ftpAccount>

Setting up a timer to clean up the system

The script defines a <timer> that cleans the system after 5 days deleting the file and removing the directory created by the script itself

                                 <timer delay="5d">
                                        <sequence>
                                               <fileDelete>
                                                     <variable name="linkFile"/>
                                               </fileDelete>
                                               <fileDelete>
                                                     <variable name="pathProduct"/>
                                               </fileDelete>
                                        </sequence>
                                 </timer>

Creating the response message

Finally the response message compliant with the SSE ICD is created using the <xml> tag

                                 <xml attributePrefix="attribute$" textTag="evaluate">
                                        <mass:returnOrderResultInputMsg xmlns:mass="http://www.esa.int/mass">
                                               <mass:commonInput>
                                                     <mass:orderId><evaluate name="orderID"/></mass:orderId>
                                               </mass:commonInput>
                                               <mass:getOrderOutput>
                                                     <mass:statusInfo>
                                                                                   <mass:statusId>0</mass:statusId>
                                                                                   <mass:statusMsg>Successful</mass:statusMsg>
                                                     </mass:statusInfo>
                                                     <mass:orderResultURL><evaluate name="orderResultURL"/></mass:orderResultURL>
                                                     <mass:userName><evaluate name="userAccount"/></mass:userName>
                                                     <mass:password><evaluate name="password"/></mass:password>
                                                     <mass:timePeriodOfService><evaluate name="timeService"/></mass:timePeriodOfService>
                                                     <mass:productSize><evaluate name="fileSize"/></mass:productSize>
                                                                            <mass:userQuota><evaluate name="availableQuota"/></mass:userQuota>
                                                                      </mass:getOrderOutput>
                                                               </mass:returnOrderResultInputMsg>
                                                        </xml>

The getQuota.xml script

The getQuota script calculates the sum of the size of the file included in the download directory

<sequence>
                              <setVariable name="usedQuota">
                                    <literal type="int"                        value="0"/>
                              </setVariable>
                              <setVariable name="subDirs">
                                    <methodInvocation methodName="listFiles">
                                           <object>
                                                  <newObject>
                                                        <classLiteral class="java.io.File"/>
                                                        <parameter>
                                                               <classLiteral class="java.io.File"/>
                                                               <variable name="downloadDir"/>
                                                        </parameter>
                                                        <parameter>
                                                               <classLiteral class="java.lang.String"/>
                                                               <variable name="name"/>
                                                        </parameter>
                                                  </newObject>
                                           </object>
                                    </methodInvocation>
                              </setVariable>
                              <setVariable name="dirIndex">
                                    <literal type="int"                        value="0"/>
                              </setVariable>
                              <while>
                                    <lt>
                                           <variable name="dirIndex"/>
                                           <arrayLength>
                                                  <variable name="subDirs"/>
                                           </arrayLength>
                                    </lt>
                                    <sequence>
                                           <setVariable name="fileList">
                                                  <methodInvocation methodName="listFiles">
                                                        <object>
                                                               <arrayLocation>
                                                                      <variable name="subDirs"/>
                                                                      <variable name="dirIndex"/>
                                                               </arrayLocation>
                                                        </object>
                                                  </methodInvocation>
                                           </setVariable>
                                           <if>
                                                  <gt>
                                                        <arrayLength>
                                                               <variable name="fileList"/>
                                                        </arrayLength>
                                                        <literal value="0"/>
                                                  </gt>
                                                  <set>
                                                        <variable name="usedQuota"/>
                                                        <plus>
                                                               <variable name="usedQuota"/>
                                                               <methodInvocation methodName="length">
                                                                      <object>
                                                                            <arrayLocation>
                                                                                   <variable name="fileList"/>
                                                                                   <literal type="int"  value="0"/>
                                                                            </arrayLocation>
                                                                      </object>
                                                               </methodInvocation>
                                                        </plus>
                                                  </set>
                                           </if>
                                           <inc>
                                                  <variable name="dirIndex"/>
                                           </inc>
                                    </sequence>
                              </while>
                              <variable name="usedQuota"/>
</sequence>

The Integration

The service has been integrated a asynchronous service for historical reasons.

In order to integrate an Asynchronous service 3 scripts are needed. In this case the first script is an empty script

<?xml version="1.0" encoding="UTF-8"?>

<sequence xmlns="http://pisa.intecs.it/mass/toolbox/xmlScript"/>

the second script is a script returning always true

<?xml version="1.0" encoding="UTF-8"?>

<literal xmlns="http://pisa.intecs.it/mass/toolbox/xmlScript" type="boolean" value="true"/>

the third script is the script described above.

The service has been integrated using the procedure reported in the Creating a Service section. The Order operation has been added using the procedure reported in the Adding an operation section.
