Listing the Service operations

When a service is imported the operation described in the export file are immediately available in the new service. When a service is created from scratch no operations are available.

To list the available operations or to create a new operation select a service then click on the Operations management link in the second row of the toolbar (a service has to be selected in order to display this link). The browse will display a table listing the available operations.

The table displayed includes the following information:

  • Operation:name of the operation. It is only a mnemonic name which is used by the TOOLBOx to store and display information related to a specific operation. It is not used to deploy the real operation and to create the WSDL file.
  • SOAP Action: it is the SOAP Action associated to the operation. The SOAP Action is used by the TOOLBOX to link the scripts to a specific SOAP message sent to a Service.
  • Type: Synchronous or Asynchronous.
  • Timeout: (only for Asynchronous operations) the timeout of the request coming from a client. It is the maximum time the TOOLBOX waits for a response from the back-end service. In practice it represents the time after that the TOOLBOX stops trying performing the Second script for a certain request.
  • Polling Rate: (only for Asynchronous. operations) indicates the rate the TOOLBOX push manager polls for the responses from the back-end service (the rate the TOOLBOX executed the second Script provided during the Operation adding).
  • Retry Attempts: (only for Asynchronous. operations) the operation can be configured such that, in case of failing pushing a response, the TOOLBOX tries to push again and again for a configurable ("Retry Attempts") amount of times.
  • Retry Rate: (only for Asynchronous. operations) the operation can be configured such that, in case of failing pushing a response, the TOOLBOX tries to push again and again with a configurable ("Retry Rate") rate.
  • Commands: It includes 3 links:
    • The Configure links allows configuring an operation
    • The Info links allows adding, displaying and updating (only for operations which are not SSE operations) the WSDL information
    • The Delete link allows deleting an operation

The "ADD Operation" button located at the end of the list allows adding a new operation.
