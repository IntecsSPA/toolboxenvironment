Complex Data

Indicates that this input shall be a complex data structure(such as a GML fragment), and provides lists of formats, encodings, and schemas supported.

The Complex Data Input in the wizard can be defined by the following interface:

  1. MaximumMegabytes: the maximum file size, in megabytes, of this input. If the input exceeds this size, the server will return an error instead of processing the inputs.

  2. Default/Supported Format: identifies the default/supported format for this input. The process shall expect input in this combination of Format/Encoding/Schema unless the Execute request specifies otherwise. Default format(mandatory) is the selected item in the Complex Data Grid.

  3. MimeType: identification of default Format for process input.

  4. Encoding: reference to default encoding for process input.

  5. Schema: reference to default XML Schema Document for process input.
