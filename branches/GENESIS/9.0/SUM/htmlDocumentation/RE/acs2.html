The Script development (2)

In the following we will describe the briefScript.xml. This script is executed by the main script when a Search SOAP message is received and the presentation tag is set to brief. The other scripts are similar and differ only in the format of the message to be sent back.

Checking the number of the returned results

The script checks the number of results returned by the database. The database does not allow retrieving "itaratorSize" results starting from "cursor". Thus the script retrieves "cursor + iteratorSize - 1" from the database. To check if the returned result set contains the requested metadata , the script checks if the number of the returned results (<dbGrtNumOfRows>) is greater or equal to the cursor.

<sequence xmlns="http://pisa.intecs.it/mass/toolbox/xmlScript">
      <setVariable name="index">
            <literal value="0"/>
      </setVariable>
      <if>
            <gte>
                  <dbGetNumOfRows>
                        <variable name="sqlResult"/>
                  </dbGetNumOfRows>
                  <atoi>
                        <variable name="cursor"/>
                  </atoi>
            </gte>

If not, a (successful) message with 0 hits is returned. The response message is created using the <xml> tag. 

      <setVariable name="responseDoc">
            <xml textTag="" attributePrefix="x">
                  <mass:processSearchOutputMsg xmlns:mass="http://www.esa.int/mass">
                  <mass:searchOutput>
                  <mass:statusInfo>
                  <mass:statusId>0</mass:statusId>
                  <mass:statusMsg>successful</mass:statusMsg>
                  </mass:statusInfo>
                  <mass:hits>0</mass:hits>
                  </mass:searchOutput>
                  </mass:processSearchOutputMsg>
            </xml>
      </setVariable>

Setting the resultset cursor

The number of elements included in the result set is NE = MIN("cursor + iteratorSize - 1", "results included in the DB"). The metadata to be returned are extracted from the results included between the "cursor" and NE. The dbNextRow foresees that to retrieve the result (row) N, the cursor has to point to the result N-1, thus at the first iteration of the "while", the cursor has to be positioned before the first row to retrieve. This operation has not to be performed if you want to start from the beginning of the resultset (cursor = 1), indeed a cursor valuing "0" is not allowed (a row numbered "0" does not exist in the database). Thus, if the cursor if greater than 1 the script sets the resultset cursor to "cursor - 1" using the <dbSetCursor> tag, otherwise the cursor is let on the first row.

      <sequence>
            <if>
                  <gt>
                        <atoi>
                              <variable name="cursor"/>
                        </atoi>
                        <literal value="1"/>
                  </gt>
                  <dbSetCursor>
                        <dec>
                              <atoi>
                                    <variable name="cursor"/>
                              </atoi>
                        </dec>
                        <variable name="sqlResult"/>
                  </dbSetCursor>
            </if>

Creating the gml file name

The gml file name is created using the <randomString> tag. The complete URL of this file will be included in the response message

      <setVariable name="gmlFileName">
            <stringCat>
                  <randomString length="8"/>
                  <string>.gml</string>
            </stringCat>
      </setVariable>
Creating the response message

The response message is created using the <xmlDocument> tag. This tag allows creating an XML Document with dynamic content. The XML message is then stored in the responseDoc variable.

The script defines the root element of the response message by means of the <element> tag: the name of the tag is defined using the <tag> tag (mass:processSearchOutputMsg); the namespaces are defined using the <attributes><namespace> tag 

      <xmlDocument>
            <element>
                  <tag>
                        <string>mass:processSearchOutputMsg</string>
                  </tag>
                  <attributes>
                        <namespace prefix="aoi" uri="http://www.esa.int/xml/schemas/mass/aoifeatures"/>
                        <namespace prefix="eoli" uri="http://earth.esa.int/XML/eoli"/>
                        <namespace prefix="mass" uri="http://www.esa.int/mass"/>
                        <namespace prefix="gml" uri="http://www.opengis.net/gml"/>
                        <namespace prefix="oi" uri="http://www.esa.int/oi"/>
                  </attributes>

the content of the root tag is defined in the <content> tag. This tag includes a fixed part and a variable part. In the fixed part some tags are set to a dummy value (mass:hits and mass:cursor) being their value not known a priori (it is the result of the execution of the variable part). The value of this tags will be set after the execution of the <xmlDocument> tag by means of the <setXMLValue> tag

Setting some variable tags

In the XML document (responseDoc) created with the <xmlDocument> tag some tags are set to a dummy value (mass:hits and mass:cursor) being their value not known a priori (it is the result of the execution of the variable part). The value of these tags is set by means of the <setXMLValue> tag

      <setXMLValue xmlns:mass="http://www.esa.int/mass">
            <variable name="responseDoc"/>
            <string>/mass:processSearchOutputMsg/mass:searchOutput/mass:hits/text()</string>
            <dec>
                  <variable name="index"/>
           </dec>
      </setXMLValue>
      <setXMLValue xmlns:mass="http://www.esa.int/mass">
            <variable name="responseDoc"/>
            <string>mass:processSearchOutputMsg/mass:searchOutput/mass:cursor/text()</string>
            <variable name="cursor"/>
      </setXMLValue>

Creating the GML file

Finally the script creates the GML file transforming the XML response using the <xslt> tag (xsl file location /home/toolbox/ACS/ACSCatalogue2GML.xsl)

      <setVariable name="gmlDoc">
            <xslt xmlOutput="true">
                  <variable name="responseDoc"/>
                  <stringCat>
                        <variable name="instDir"/>
                        <string>ACSCatalogue2GML.xsl</string>
                  </stringCat>
                  <parameter name="hostName">
                        <variable name="HTTP_BASE_PATH"/>
                  </parameter>
            </xslt>
      </setVariable>

The file (created and stored in the gmlDoc variable) is then processed with a JAVA method ("adjust", which is included in the it.intecs.pisa.soap.toolbox.services.util.gmlAdjuster class) and finally stored on a predefined directory. This function performs some operations that can not be performed using the xsl language. Moreover it converts a polygon compliant with the EOLI framework into a polygon which is compliant with the SSE framework.

      <dumpXML>
            <stringCat>
                  <variable name="gmlBasePath"/>
                  <variable name="gmlFileName"/>
            </stringCat>
            <methodInvocation methodName="adjust">
                  <class>
                        <classLiteral class="it.intecs.pisa.soap.toolbox.services.util.gmlAdjuster"/>
                  </class>
                  <parameter>
                        <classLiteral class="org.w3c.dom.Document"/>
                        <variable name="gmlDoc"/>
                  </parameter>
            </methodInvocation>
      </dumpXML>
