Example of a WPS processing

In the following an example of a Describe Process and the related Grass script is described. The following example aims to describe the work that shall be done in order to provide to Toolbox all needed information.
The first part that shall be provided is the DescribeProcess document, which describes the input and output data types for each processing.
The ProcessDescription data structure, returned by a Describe Process, shall specify the model  data type plus the multiplicity and use of each listed part in the operation response. The multiplicity and use columns in the following tables specify the optionality of each listed parameter and data structure in the DescribeProcess operation response.
Each parameter is described by a data structure that specifies the allowable formats, encoding, and units of measure (when applicable). For each input parameter, the process can indicate that it needs one of the following:
  1. ComplexData (such as XML or imagery), in one of the following allowable combinations of format (mimetype, encoding, and schema). The value of this complex data structure can be (either) directly encoded in the Execute operation request or made available through a web accessible URL.
  2. LiteralData, with a specified data type, allowable values, default value, and allowable unit of measure indicated.
  3. BoundingBoxData, using one of the supported coordinate reference systems.
For each output parameter, the process can indicate similar information about the corresponding forms of output parameters. Again, there are three types of process outputs: ComplexOutput, LiteralOutput, and BoundingBoxOutput.
More precisely, a response from the DescribeProcess operation shall be a ProcessDescriptions data structure that includes one or more ProcessDescription data structures, as listed in Table 15 in the document OGC05-007r7, downloadable from OGC web site
The above mentioned OGC document is the reference document and describes deeply all relations between the Describe Process document  the WPS logic.

Depending on the processing method, a GRASS or a Toolbox script shall be provided in order to define the processing logic.. In the following a Grass script example is provided. This script can be used with the above Describe Process example in order to test all WPS mechanisms implemented. The main thing to note is that the input data is provided to the script in the form of variables (shell variables in the script below) following a predefined syntax. The output of the processing ( to be returned to the client) follows the same concepts. The template script, provided by default by Toolbox, provides a detailed description about the input/outptu variables.
