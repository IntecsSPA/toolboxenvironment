Configuration procedures

Toolbox Archiving Service

ARMS Servlet

The ARMS servlet can be configured by modifying its properties fils and restarting the servlet. In order to do this, follow these steps:
  • Access Tomcat manager application and stop the ARMS servlet. You can stop the Tomcat instance instead. See Tomcat manual for the a description of the procedure.
  • Open the properties file that can be found under <TOMCAT_HOME>/webapps/ArchivingServer/WEB-INF/classes/config.properties
  • Modify the properties. Save the properties file.
  • Restart the servlet or the Tomcat instance (depending on what has been done in step 1)
The table below describes all properties parameters that can be changed in the ARMS servlet configuration. For each one a description is given.

Configuration key

Type

Description
download.dir Path to directory This parameter provides a full path to a diretory where all data and metadata is temporarily downloaded
http.public.url URL Full URL that shall be used for accessing the ARMS servlet. E.g. http://localhost:8080/ArchivingServer This parameter shall be used when creating links (e.g. data download links)
delete.after Complex type This parameter provides a time interval after which data shall be automatically deleted. This value shall be expressed concatenating a number and one of the following letters:
  • s for seconds
  • m for minutes
  • h for hours
  • d for days
  • w for weeks
fail.dorollback Boolean This parameter activates or deactivates the rollback procedure in case of error.
publish.local.http.enable Boolean This parameter activates or deactivates the HTTP publish feature. This parameter has an higher priority respect to the parameter provided by the input message
publish.local.ftp.enable Boolean This parameter activates or deactivates the FTP publish feature. This parameter has an higher priority respect to the parameter provided by the input message
publish.local.ftp.rootdir Path to directory This parameter provides a full path to a directory where all FTP data is stored.
publish.local.ftp.ip IP Address This parameter provides the binding address for the internal FTP server.
publish.local.ftp.port Number This parameter provides the port associated to the FTP server.
publish.local.folder.interval Complex Type This parameter provides a time interval between directory checks. At every check new file is searched in the directory and automatically deployed into the ARMS. This value shall be expressed concatenating a number and one of the following letters:
  • s for seconds
  • m for minutes
  • h for hours
  • d for days
  • w for weeks
publish.local.folder.dir Full path to directory This parameter provides a full path to a directory that will be checked for data deployment.
publish.geoserver.workspace Stirng This parameter provides the Geoserver workspace name under which all data will be deployed.

Toolbox Archiving Service

The Toolbox Archiving Service can be deployed with few steps by following a wizard like interface. Once deployed the service can be configured as all of the Toolbox services. The Toolbox administration web pages allow the administrator to fine tune the configuration in order to match his needs. In order to do this, please follow the steps below:
  • Login into the Toolbox Administration web pages.
  • Select the Archiving Service from the drop down in the top left.
  • Click on "Service Configuration" link, then on "Change the configuration" link.
  • Update the cariables in the "Configure service variables" section. A description is done below for each parameter.
  • Click on the "Configure" button.

Variable

Type

Description
Allow publishing data via WMS,WFS and WCS Boolean This parameter enables/disable publishing data through the WMS,WCS and WFS protocols.
Allow publishing data via FTP Boolean This parameter enables/disable publishing data through the FTP protocol.
Allow publishing data via HTTP Boolean This parameter enables/disable publishing data through the HTTP protocol.
Url of the Geoserver instance URL This parameter provides the URL of the Geoserver instance to be used to publish data.
Url of the catalogue instance URL This parameter provides the URL of the ebRIM catalogue instance to be used to store metadata.
Url of the archiving server URL This parameter provides the URL of the Archiving Servlet used for performing all tasks.
