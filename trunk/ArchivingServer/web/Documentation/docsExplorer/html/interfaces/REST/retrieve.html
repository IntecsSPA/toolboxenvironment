Retrieve operation

This REST endpoint will allow a client to retrieve all links to services where data and metadata is deployed and thus is accessible. This operation is also used to retrieve the status of a store operation that has been successfully committed. For a full description of how this operation works, please refer to this section The retrieve operation is accessible at the URL http://<TOMCAT_IP>:<TOMCAT_PORT>/ArchivingServer/service/retrieve/<ITEMD_ID>
The HTTP method to use for accessing the operation is GET

Input message

The only parameter needed by the retrieve operation is the identifier through which data and metadata can be identified. This identifier is provided by the <ITEMD_ID> field specified in the endpoint URL.

Output message

The response message is a complex JSON object which contains all links to data and metadata and a status field. Its structure is described in the table below.

Field

Type

Mandatory

Description
success Boolean Yes This field specifies if retriving data and metadata links has been successfull.
id Integer Yes This field contains the Item Id
downloadStatus String Yes This field contains the status of the store operation. Valid values are:
  • DOWNLOADING
  • PUBLISHED
http Array No This field contains all links to data deployed through HTTP protocol. Each field of the array is a string containing the URL to data
ftp Array No This field contains all links to data deployed through FTP protocol. Each field of the array is a string containing the URL to data
catalogues Array No This field contains all links to ebRIM catalogues where metadata has been harvested. Each field of the array is the URL through which the SOAP interface of the catalogue can be accessed.
geoserver Array No This field contains all links to Geoserver instances where spatial data has been deployed. Each field of the array is the URL through which the Geoserver instance can be accessed.
