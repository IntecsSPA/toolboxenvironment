Cleanup

Tag Name addCleanupMarker
Attributes
Name Type Use Default Description
value xsd:string required The value of the marker to add.
Description It adds a clean up marker whose value is given by the "value" attribute: in case of failure of the script the related clean up operation(s) shall be executed.
Examples 
<addCleanupMarker value="A_Marker">

Adds the marker "A_Marker".

Tag Name cleanupProcedure
Description It encloses the list of operations to be performed in case of failure of the script. E
Examples 
<cleanupProcedure>
	<cleanup>
		<fileDelete>
			<string>file.txt</string>
		</fileDelete>
	</cleanup>
</cleanupProcedure>

In case of failure of the script deletes (without checking the existen markers) the file "file.txt". 

<cleanupProcedure>
	<cleanup marker="Marker_One">
		<!-- cleanup script related to the marker Marker_One -->
	</cleanup>
	<cleanup marker="Marker_Two">
		<!--cleanup script related to the marker Marker_Two-->
	</cleanup>
	<cleanup marker="Marker_Three">
		<!--cleanup script related to the marker Marker_Three-->
	</cleanup>
</cleanupProcedure>
In case of failure of the script checks if the marker "Marker_One" exsists: if so, the first cleanup script is executed. The same procedure is performed for the other two cleanup scripts. 
<cleanupProcedure>
	<cleanup>
		<!-- cleanup script -->
	</cleanup>
	<cleanup marker="Marker_One">
		<!-- cleanup script related to the marker Marker_One -->
	</cleanup>
	<cleanup marker="Marker_Two">
		<!--cleanup script related to the marker Marker_Two-->
	</cleanup>
	<cleanup marker="Marker_Three">
		<!--cleanup script related to the marker Marker_Three-->
	</cleanup>
</cleanupProcedure>
In case of failure of the script, executes the first cleanup script (without performing any check on the existent markers), then checks if the marker "Marker_One" exsists: if so, the second cleanup script is executed. The some procedure is performed for the other two cleanup scripts.

Tag Name removeCleanupMarker
Attributes
Name Type Use Default Description
value xsd:string required The value of the marker to remove.
Description It removes a clean up marker whose value is given by the "value" attribute: in case of failure of the script the related clean up operation(s) shall NOT be executed.
Examples 
<removeCleanupMarker value="A_Marker">

Removes the marker "A_Marker".
