Creating the TOOLBOX ADMINISTRATOR account

The TOOLBOX is released with a TOOLBOX ADMINISTRATOR Web application account having the following parameters:

  • username="admin"
  • password="admin"

Even though you can choose to keep and use this account, its deletion is strongly recommended in order to prevent unauthorized accesses.

One or more administrator accounts can be created.

To create a new account, open the adminUsers.xml file under: Root_Directory_Of_Tomcat/TOOLBOX/WEB-INF/xml/ with any text/XML editor. You will find its content is: 

<?xml version="1.0" encoding="UTF-8"?>
<adminUsers xmlns="http://pisa.intecs.it/mass/toolbox/adminUsers">
   <adminUser userName="admin" password="admin"/>
</adminUsers>
In order to substitute the release account with a new one, simply replace the attribute values of userName and password of the </adminUser> tag with new values:
<?xml version="1.0" encoding="UTF-8"?>
<adminUsers xmlns="http://pisa.intecs.it/mass/toolbox/adminUsers">
   <adminUser userName="userDefinedUserName" password="userDefinedpassword"/>
</adminUsers>

In order to add an account, copy and paste the element <adminUser userName="userDefinedUserName" password="userDefinedpassword"/> and replace "userDefinedUserName" and "userDefinedpassword" with new values: 

<?xml version="1.0" encoding="UTF-8"?>
<adminUsers xmlns="http://pisa.intecs.it/mass/toolbox/adminUsers">
   <adminUser userName="userDefinedUserName" password="userDefinedpassword"/> 
   <adminUser userName="anotherUserDefinedUserName" password="anotherUserDefinedpassword"/> 
</adminUsers>

Be careful not to change the file structure during these operations. This XML file is validated by the TOOLBOX ADMISTRATOR Web application according to a schema. If the structure changes an Exception is thrown when attempt to login and there is no other way to proceed until the adminUsers.xml file is valid.
